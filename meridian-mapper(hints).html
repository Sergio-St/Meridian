What is Meridian?
Meridian is an AI-first tool that maps any medical term — formal or informal, coded or free text — to the correct OMOP standard concept. It's designed for data engineers, epidemiologists, and clinical informatics teams who build OMOP pipelines.
🏥
What is OMOP?
OMOP (Observational Medical Outcomes Partnership) is a global standard for organizing health data. It lets researchers combine patient data from hospitals, pharmacies, and insurers worldwide using a common language.
🗺️
What is concept mapping?
Every hospital uses its own codes and terms for diseases, drugs, and tests. Mapping converts these into OMOP standard concepts — like translating medical records from dozens of local dialects into one universal language.
⚡
Why does it take so long today?
The current standard tool (Usagi) uses string-matching and fails on informal terms. A team of specialists reviews thousands of terms manually. A single dataset takes 3–6 months and costs $100k–500k.
🤖
How does Meridian fix this?
Meridian uses a large language model to understand medical language contextually. "Sugar diabetes" → Type 2 diabetes mellitus (SNOMED 44054006). "Bad cholesterol" → LDL cholesterol (LOINC 13457-7). Instantly.

How the AI mapping works
1

Semantic interpretation

The LLM reads your source term in context, understanding informal language, abbreviations, brand names, and locally invented codes.

2

Vocabulary search

The system queries the OHDSI Athena vocabulary database + semantic vector search across 10M+ OMOP concepts to find candidates.

3

Ranked candidates + reasoning

The AI returns the best match with a confidence score and plain-English reasoning, plus 1–3 alternatives with scores.

4

Human review for low-confidence

Terms below 80% confidence are flagged for specialist review. Your corrections are logged and feed back into model improvement.

⚠ Meridian is a decision-support tool. All mappings intended for regulatory submissions should be reviewed by a qualified clinical informaticist before use in production.

Meridian exposes a simple REST API for integrating AI-powered OMOP mapping into your ETL pipelines, data quality workflows, or clinical research tools.

Quick start
Send a POST request to https://api.meridian.health/v1/map with your terms:
# Map three terms with a single API call curl -X POST https://api.meridian.health/v1/map \ -H "Authorization: Bearer YOUR_API_KEY" \ -H "Content-Type: application/json" \ -d '{ "terms": ["sugar diabetes", "HbA1c elevated", "metformin 500mg"], "options": { "min_confidence": 80, "max_alternatives": 3 } }'
Recommended workflow
The typical integration pattern for an OMOP ETL pipeline:
1

Extract unique source terms

From your source database, extract a deduplicated list of all unique values from columns you need to map (diagnosis codes, drug names, lab names, etc.).

2

Send to Meridian API in batches

Send up to 100 terms per request. For large datasets (1M+ rows), deduplicate first — typically 95% of rows share a small set of unique terms.

3

Auto-approve high-confidence mappings

Mappings with confidence ≥ 80 can be automatically applied. Flag lower-confidence terms for your clinical reviewer queue.

4

Apply mapping table to full dataset

Join your source data against the approved mapping table to populate OMOP concept_id fields in the target CDM tables.

5

Run OMOP data quality checks

Use OHDSI's DataQualityDashboard to validate the result. Meridian exports a QA report showing coverage and unmapped terms.

✓ This workflow typically reduces manual review time from weeks to hours for a standard clinical dataset.

Python integration
import requests import pandas as pd MERIDIAN_API_KEY = "mer_live_xxxxxxxxxxxx" BASE_URL = "https://api.meridian.health/v1" def map_terms(terms: list[str], min_confidence: int = 80) -> list[dict]: """Map source terms to OMOP concepts via Meridian API.""" results = [] # Process in batches of 100 for i in range(0, len(terms), 100): batch = terms[i:i+100] resp = requests.post( f"{BASE_URL}/map", headers={"Authorization": f"Bearer {MERIDIAN_API_KEY}"}, json={"terms": batch, "min_confidence": min_confidence} ) resp.raise_for_status() results.extend(resp.json()["results"]) return results # Example: map all unique diagnosis values from your source table source_df = pd.read_csv("source_diagnoses.csv") unique_terms = source_df["diagnosis_text"].unique().tolist() mappings = map_terms(unique_terms, min_confidence=80) mapping_df = pd.DataFrame(mappings) # Join back to source data result = source_df.merge( mapping_df[["source_term", "concept_id", "concept_name", "confidence"]], left_on="diagnosis_text", right_on="source_term", how="left" ) result.to_csv("omop_mapped_diagnoses.csv", index=False) print(f"Mapped {len(mapping_df)} unique terms → OMOP concept IDs")
